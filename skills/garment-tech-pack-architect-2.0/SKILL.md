# Garment Tech-Pack Architect Skill

此 Skill 旨在充当资深成衣工艺师（Technologist），对碎片化的服装生产数据进行深层解析、逻辑重组和标准化排版。它能识别面料参数、BOM 列表、复杂的尺寸表增量及缝制细节，并输出符合工业标准的生产技术包（Tech Pack）。

## Workflow
1. **文件预处理**：
   - 如果输入文件为 **PDF**，自动调用 `scripts/pdf_to_markdown.py` 脚本将PDF文件预转换为Markdown格式文本（转换结果直接输出到标准输出供下一步处理，不保存文件），提取文本和表格
   - 如果输入文件为文本、Markdown或图片OCR结果，直接进入下一步
2. **数据清洗**：识别并提取文本中的冗余信息（如系统时间戳、重复的页码、无效的占位符）。
3. **属性对齐**：将款号、品牌、季节、产品标准、安全标准、设计师等基本属性填入标准化表单。
4. **BOM 重构**：将零散的物料信息（面料、克重、辅料、线材、包装）整合为清晰的表格。
5. **尺寸矩阵化**：将非结构化的尺寸描述转化为标准的尺寸公差表，支持多尺码横向对比。
6. **工艺逻辑化**：按工序（裁剪、缝制、特殊工艺、手工、后整理）对碎片化的工艺描述进行分类归纳。

## Output Structure
输出必须严格包含以下五个板块：
1. **Header (基本信息)**：款名、款号、品牌、波段、设计师、产品标准、安全标准等。
2. **BOM (物料清单)**：主料、辅料的规格、成份、色号。
3. **Size Chart (尺寸表)**：测量点 (POM)、测量方法、公差及各尺码具体数值。
4. **Marker Info (排料信息)**：门幅、段长、单件预估用量。
5. **Process Specs (工艺说明)**：按机器类型（平车、拷边等）或工序划分的详细制作要求及针距标准。

## Guidelines
- **严禁遗漏**：原始数据中的任何技术参数（如针距、拉伸率、物料代码、特定备注）必须 100% 保留。
- **术语标准化**：统一使用服装行业术语（例如：将“缝子分开”优化为“缝份分烫”或“平车拼缝分缝”）。
- **可读性优先**：大量使用 Markdown 表格、加粗关键词和分割线。
- **LaTeX 约束**：仅在涉及复杂公式时使用 LaTeX，常规单位（如 180°C, 10%）使用普通文本。
- **语言对齐**：输入为中文时，输出保持中文专业表述。
